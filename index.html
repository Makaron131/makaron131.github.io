788. 逆序对的数量

发表于 2022-04-17 分类于 acwing

题解

关键在于，当归并的过程中，左区间的数比右区间的数大时，
左区间 剩余的数 都可以和 右区间 当前的数，组成逆序对。

所以当左区间数比右边的大时，加一下左区间剩余的数的数量。
最终就能得到总逆序对了。

坑点：res 要用 long long，printf 要写成 %lld

说明

比较用的基准值 x 可以怎么取？

arr[l], arr[r], arr[l + r >> 1] 都可以

i、j 两个边界的起点为什么要 + 1 和 - 1

由于每次两两比较完后，i 和 j 都要各前进一格，这里为了写起来好看，就设置了这样的初始值配合 do while 来实现。

为什么要 i < j 才交换

如果 i >= j，说明基准值 x 的两边都有序了，不应该再交换了。

递归时中间边界可以怎么写？

i - 1 和 i，这时 x 不能取到 arr[l]，不然会有边界问题。
同理 j 和 j + 1，x 就不能取 arr[r]。
例子就是排 1 2 这样的数组时会死循环。
另外也别想着写 i 和 j，也会死循环。

如何让快排稳定？

快排时多一个参数加进来排序，如初始下标之类的。
反正就是说，如果数组里每个数都不一样，那么快排不就稳定了吗？
